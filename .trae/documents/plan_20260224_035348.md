## INSERT语句合并优化方案

### 问题分析

当前INSERT语句是逐条生成的，格式为：
```sql
INSERT INTO `schema`.`table`(`col1`,`col2`,...) VALUES('val1','val2',...);
INSERT INTO `schema`.`table`(`col1`,`col2`,...) VALUES('val3','val4',...);
```

需要合并为：
```sql
INSERT INTO `schema`.`table`(`col1`,`col2`,...) VALUES('val1','val2',...),('val3','val4',...);
```

### 修复方案

#### 1. 创建OptimizeInsertSqls函数

在`actions/sql_optimize.go`文件中添加新函数：

```go
// OptimizeInsertSqls 优化 INSERT 语句，将相同表的多条 INSERT 语句合并为单条 VALUES 多组数据的格式
func OptimizeInsertSqls(sqls []string, maxSqlSize int, fixTrxNum int) []string
```

主要逻辑：
- 使用正则表达式解析INSERT语句，提取表名、列名列表和VALUES部分
- 按`schema.table.列名列表`分组聚合VALUES
- 合并时检查文件大小限制（1MB）
- 确保数据唯一性，避免重复数据

#### 2. 修改processBatch函数

在`actions/table_index_dispos.go`文件的`processBatch`函数中：
- 在处理INSERT语句时调用`OptimizeInsertSqls`函数
- 保持DELETE语句的优化逻辑不变

#### 3. 关键实现细节

- **正则表达式**：`(?i)^INSERT\s+INTO\s+\x60([^\x60]+)\x60\.\x60([^\x60]+)\x60\(([^)]+)\)\s+VALUES\s*\(([^)]+)\);$`
- **大小限制**：合并后的单条INSERT语句不超过1MB
- **数据唯一性**：使用map去重，确保不会产生重复数据
- **格式保留**：保持原有的表结构、字段顺序和数据类型不变
- **特殊字段处理**：保留DATE_FORMAT等函数的格式转换逻辑

2. 在`processBatch`函数中调用`OptimizeInsertSqls`函数
